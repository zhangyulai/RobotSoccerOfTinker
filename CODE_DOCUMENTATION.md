# Competition 目录代码文档

本文档详细说明了 `Unity-RL-Playground/Assets/Competition/Scripts/` 目录下所有C#脚本文件的功能和用途。该目录专注于机器人竞赛场景，主要包括足球比赛、寻球任务和站立控制等竞技性任务。

## 目录概览

Competition目录包含以下核心组件：

- **游戏管理**: `GameManager.cs` - 负责比赛规则、计分和场景管理
- **目标检测**: `GoalTrigger.cs` - 检测进球事件
- **机器人代理**: 多种Tinker机器人强化学习代理
- **基础运动**: `Tinker1Agent.cs` - 基础行走控制
- **寻球任务**: `TinkerSeekBallAgent.cs` - 专门寻找足球的代理
- **完整足球**: `TinkerSeekBallAndGoalAgent.cs` - 完整的足球比赛代理（寻球+进球）
- **站立恢复**: `TinkerStandUp.cs` - 从倒地状态恢复站立的代理

## 核心脚本详解

### GameManager.cs
**功能**: 足球比赛游戏管理器

**主要职责**:
- **计分系统**: 管理Blue队和Purple队的比分，显示在UI上
- **碰撞检测**: 使用边界框重叠检测判断球是否进入球门
- **场景复位**: 进球后自动复位球和两个机器人到初始位置
- **音效管理**: 
  - 背景音乐循环播放
  - 进球时播放欢呼音效
  - 比赛结束时播放胜利音效
- **胜利判定**: 先得3分的队伍获胜，显示对应颜色的胜利图片
- **重新开始**: 提供Start按钮，可以重置比赛重新开始

**关键技术特点**:
- 使用`TeleportRoot`方法精确复位机器人位置和姿态
- 防止重复计分的标记系统（`hasScoredBlue`/`hasScoredPurple`）
- 专门的背景音乐AudioSource和音效AudioSource分离
- 支持玩家1和玩家2两个机器人的独立控制

### GoalTrigger.cs
**功能**: 进球触发器

**主要职责**:
- **触发检测**: 使用Unity的`OnTriggerStay`检测球与球门的接触
- **事件通知**: 当检测到进球时，调用对应机器人代理的`OnGoalScored()`方法
- **标签验证**: 只对Tag为"ball"的物体进行处理

**使用场景**:
- 附加在球门Collider上，作为进球检测的辅助组件
- 与`TinkerSeekBallAndGoalAgent.cs`配合使用

### Tinker1Agent.cs
**功能**: Tinker机器人基础运动代理

**主要特点**:
- **基础控制**: 实现Tinker机器人的基本行走、转向控制
- **键盘输入**: 支持方向键和WASD两种控制模式
- **强化学习**: 基于ML-Agents的PPO算法训练
- **多实例训练**: 自动克隆24个实例进行并行训练
- **奖励函数**: 
  - 生存奖励（保持平衡）
  - 方向控制奖励（跟踪目标速度和角速度）
  - 姿态稳定性奖励

**适用场景**: 
- 作为其他复杂任务的基础运动能力
- 单独的行走控制训练

### TinkerSeekBallAgent.cs
**功能**: 寻球任务专用代理

**主要特点**:
- **球追踪**: 专门优化用于寻找和接近足球
- **增强观测**: 包含球的相对位置、速度、距离等丰富信息
- **改进奖励函数**:
  - **前进奖励**: 鼓励机器人实际移动（解决"躺平"问题）
  - **距离奖励**: 离球越近奖励越高
  - **方向奖励**: 鼓励正面朝向球
  - **接触奖励**: 非常接近球时给予额外奖励
  - **犹豫惩罚**: 在球旁边徘徊会被轻微惩罚
- **时间加速**: 训练时支持20倍时间加速提高效率

**技术亮点**:
- 解决了传统RL中机器人不愿移动的问题
- 通过精心设计的奖励函数引导有效行为
- 包含能量效率惩罚防止过度动作

### TinkerSeekBallAndGoalAgent.cs
**功能**: 完整足球比赛代理（寻球+进球）

**主要特点**:
- **双重任务**: 同时处理寻球和向球门带球两个阶段
- **状态感知**: 能够检测是否已经"持球"（球在脚附近）
- **阶段奖励**:
  - **寻球阶段**: 与`TinkerSeekBallAgent`类似
  - **持球奖励**: 成功接触球时获得5分大奖励
  - **带球向门**: 鼓励持球状态下向球门移动
  - **进球奖励**: 成功进球获得20分大奖励
- **智能复位**: 进球后自动将球随机放置在新位置
- **防摔倒策略**: 摔倒惩罚降低，鼓励大胆试错

**关键技术**:
- **持球检测**: 通过脚部与球的距离判断是否持球
- **动态奖励调整**: 根据是否持球调整前进奖励策略
- **安全速度控制**: 接近球时限制速度防止冲过头

### TinkerStandUp.cs
**功能**: 站立恢复代理

**主要特点**:
- **倒地恢复**: 专门训练机器人从各种倒地姿态恢复站立
- **随机初始化**: 每次episode开始时随机设置机器人的初始姿态
- **键盘控制模式**: 支持手动控制测试
- **多实例训练**: 训练时创建32个实例，并行训练
- **特殊奖励函数**: 
  - 主要基于`UpDotVertical`（身体up向量与世界up向量的点积）
  - 身体越竖直，奖励越高
  - 动作幅度惩罚防止过度动作

**应用场景**:
- 作为足球比赛中摔倒后的恢复策略
- 提高机器人在复杂环境中的鲁棒性

## 竞赛系统架构

### 整体工作流程
1. **初始化**: `GameManager`设置比赛场景，加载机器人和球
2. **控制循环**: 机器人代理通过强化学习或键盘输入控制运动
3. **目标检测**: `GoalTrigger`持续检测进球事件
4. **计分更新**: 进球后`GameManager`更新比分并播放音效
5. **场景复位**: 进球后自动复位球和机器人位置
6. **胜利判定**: 达到3分后显示胜利画面

### 技术栈特点
- **Unity ML-Agents**: 所有代理都基于ML-Agents框架
- **ArticulationBody**: 使用Unity的ArticulationBody系统进行物理模拟
- **强化学习算法**: 主要使用PPO（Proximal Policy Optimization）
- **多智能体**: 支持两个机器人同时参与比赛
- **实时交互**: 支持键盘实时控制和AI自动控制切换

### 训练策略
- **并行训练**: 每个代理自动克隆多个实例加速训练
- **时间加速**: 训练时可启用20倍时间加速
- **奖励塑形**: 精心设计的奖励函数解决稀疏奖励问题
- **课程学习**: 从简单任务（站立）到复杂任务（足球）逐步训练

## 使用指南

### 开发者使用
1. **基础运动**: 从`Tinker1Agent.cs`开始，确保基础行走正常
2. **寻球任务**: 使用`TinkerSeekBallAgent.cs`训练寻球能力
3. **完整比赛**: 集成`TinkerSeekBallAndGoalAgent.cs`实现完整足球功能
4. **鲁棒性**: 添加`TinkerStandUp.cs`处理摔倒恢复

### 参数调整
- **奖励系数**: 根据训练效果调整各类奖励的权重
- **关节参数**: 修改`kb`数组调整各关节的控制灵敏度
- **时间步长**: `Time.fixedDeltaTime = 0.01f`确保物理模拟稳定
- **训练实例数**: 根据硬件性能调整并行训练数量

### 调试技巧
- **键盘控制**: 启用`keyboard = true`进行手动测试
- **时间加速**: 训练时启用`accelerate = true`提高效率
- **观测检查**: 通过`CollectObservations`方法验证输入数据正确性
- **奖励分析**: 监控各类奖励分量，确保训练方向正确

这个竞赛系统提供了一个完整的机器人足球比赛平台，涵盖了从基础运动到高级策略的所有必要组件。
